1. Schema Design Philosophy

Since we are using MongoDB with Beanie (ODM), we leverage:

Embedded Documents: For tightly coupled data (e.g., specific constraints on a Task).

References (Links): For loose coupling (e.g., Linking a Transaction to a Project).

Vector Search: Native embeddings stored directly on the Task and Journal documents.

2. Core Collections

2.1. Users (users)

Stores the "Constitution" and Seasonality settings.

class SeasonConfig(BaseModel):
name: str # e.g., "Q1 2026: Health First" # Weights for Agent Decision Making (0.0 - 1.0)
weight_health: float = 0.3
weight_wealth: float = 0.2
weight_career: float = 0.3
weight_joy: float = 0.1
weight_learning: float = 0.1

class UserSettings(BaseModel):
daily_work_hours_cap: int = 8
currency_symbol: str = "₹"
timezone: str = "Asia/Kolkata"
theme: str = "dark"

class User(Document): # \_id is auto-generated by Mongo
clerk_id: str # Indexed, Unique
email: str
display_name: str

    settings: UserSettings
    current_season: SeasonConfig

    # Resilience Tracking
    last_login_at: datetime
    current_streak: int = 0

    class Settings:
        name = "users"
        indexes = [
            IndexModel("clerk_id", unique=True)
        ]

2.3. Projects (projects) - Layer 3

The Tactical container.

class ProjectStatus(str, Enum):
ACTIVE = "active"
PAUSED = "paused"
COMPLETED = "completed"
ICEBOX = "icebox"

class Project(Document):
user_id: str
theme_id: Optional[PydanticObjectId] # Link to Theme (Optional)

    title: str
    description: str
    status: ProjectStatus = ProjectStatus.ACTIVE

    # The Golden Triangle Constraints
    deadline: Optional[datetime]
    budget_financial_cap: float = 0.0
    budget_energy_cap: int = 0 # Total estimated 'story points'

    created_at: datetime = Field(default_factory=datetime.now)

    class Settings:
        name = "projects"

2.4. Tasks (tasks) - Layer 1

The execution unit. Includes vector embeddings for "Task Similarity" search.

class TaskStatus(str, Enum):
TODO = "todo"
IN_PROGRESS = "in_progress"
DONE = "done"
BACKLOG = "backlog"

class Task(Document):
user_id: str
project_id: Optional[PydanticObjectId] # Link to Project

    title: str
    context: Optional[str] # e.g., "At Desk", "Errand"

    # Scheduling
    status: TaskStatus = TaskStatus.BACKLOG
    scheduled_date: Optional[date]
    scheduled_time_block: Optional[str] # "09:00-11:00"

    # The 3 Costs
    estimated_minutes: int
    energy_cost: int = Field(ge=1, le=10)
    financial_cost: float = 0.0

    # Resilience
    is_overdue: bool = False
    overdue_count: int = 0

    # Semantic Memory
    embedding: Optional[List[float]] # 768-dim vector (Gemini)

    class Settings:
        name = "tasks"
        indexes = [
            IndexModel("user_id"),
            IndexModel("scheduled_date"),
            # Vector Index definition would go here in Atlas setup
        ]

2.6. Journals (journals) - Feedback Loop

Stores structured feedback + unstructured text for RAG.

class JournalLevel(str, Enum):
DAY = "day"
MONTH = "month"
YEAR = "year"

class Journal(Document):
user_id: str
level: JournalLevel
date_ref: date

    # Metrics
    sentiment_score: int # 1-10
    energy_score: int
    productivity_score: int

    # Text Data
    content: str # "I felt tired because..."
    blockers: Optional[str]
    wins: Optional[str]

    # Semantic Memory
    embedding: Optional[List[float]]

    class Settings:
        name = "journals"

class Budget(Document):
